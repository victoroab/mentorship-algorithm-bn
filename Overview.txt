Mentoring System 

--- Usage -> A department in a university

Brief
-Users -> Students (Mentees), Sudents (Mentors)

-Functionality Summary
 1. Relationships (1**3) (Mentor, Mentee)
 2. Registration (Bio-information, picture upload, transcript?, preferences(football, etc), course, skills, availability...)
 3. Login (Email , Password)
 4. Selection Process (Manual, Automatic - System picks based on matching algorithm)

-Additional Functionality
 1. Passord reset
 2. Email Notifications


-Technologies
 1. React
 2. Typescript
 3. Github / Git
 4. Node js
 5. Express js
 6. Postgresql

-Architecture
 1. Client-Server
 2. Monorepo
